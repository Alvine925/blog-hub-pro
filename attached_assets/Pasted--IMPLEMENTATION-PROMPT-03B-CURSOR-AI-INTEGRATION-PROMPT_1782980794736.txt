# IMPLEMENTATION PROMPT 03B

# CURSOR AI INTEGRATION PROMPT GENERATOR

## Objective

Implement a **Cursor Prompt Generator** that automatically creates a high-quality implementation prompt for Cursor AI.

The generated prompt should allow a developer to copy it directly into Cursor and have Cursor integrate the user's existing website with the Lunar CMS REST API.

Do **NOT** generate generic prompts.

The prompt must be dynamically generated using the user's workspace, selected framework, API configuration and selected content.

---

# Existing Functionality

The following already exists and MUST NOT be rebuilt.

* Workspace
* Authentication
* REST API
* API Keys
* Blogs
* Pages
* Collections
* Categories
* Media
* Existing Dashboard
* Existing UI Components

Reuse all existing functionality.

---

# Goal

A user should complete the following workflow.

```text
Generate API Key

↓

Open Integration Center

↓

Choose Cursor

↓

Choose Framework

↓

Choose Content

↓

Generate Prompt

↓

Copy Prompt

↓

Paste into Cursor

↓

Cursor automatically integrates the website
```

---

# Prompt Generator

Create

```text
CursorPromptGenerator.ts
```

Responsibilities

* Load the Cursor prompt template.
* Replace placeholders.
* Inject the selected API endpoints.
* Inject authentication details.
* Inject selected content types.
* Return a complete implementation prompt.

Example structure

```text

Do not rebuild the project.

Inspect the existing codebase before making changes.

Reuse existing architecture.
```

# Authentication Flow

The generated prompt should explain

Every request should include

```http
Authorization:
Bearer API_KEY
```

Example

```typescript
fetch(
`${CMS_API_URL}/blogs`,
{
headers:{
Authorization:`Bearer ${CMS_API_KEY}`
}
})
```

Never expose Secret Keys inside frontend projects.

Only Publishable Keys should be used in browser applications.

# Acceptance Criteria

The generated Cursor prompt is complete when:

* It is personalized using the selected framework and workspace.
* It contains the correct API base URL and authentication method.
* It instructs Cursor to inspect and reuse the existing project.
* It avoids duplicate architecture.
* It generates production-ready code.
* It does not require the developer to manually explain the API.
* The prompt can be copied directly into Cursor and used without further editing.
