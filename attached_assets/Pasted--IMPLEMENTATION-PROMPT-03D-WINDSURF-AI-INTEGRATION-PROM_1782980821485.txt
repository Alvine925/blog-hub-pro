# IMPLEMENTATION PROMPT 03D

# WINDSURF AI INTEGRATION PROMPT GENERATOR

## Objective

Implement a **Windsurf Prompt Generator** that produces highly structured prompts optimized for Windsurf's agent-based development workflow.

Unlike other AI coding tools, Windsurf performs best when given a clear execution plan, explicit boundaries, reusable architecture guidelines, and validation steps. Every generated prompt should guide Windsurf through inspecting the project, planning the implementation, making incremental changes, and validating the final result.

The generated prompt should allow a user to copy it directly into Windsurf without additional editing.

---

# Existing Functionality

The following functionality already exists and MUST NOT be rebuilt.

* User Authentication
* Workspace Management
* Dashboard
* API Keys
* REST API
* Blogs
* Pages
* Categories
* Tags
* Events
* Media Library
* Existing Database
* Existing UI Components

The generated prompt must instruct Windsurf to extend the existing project rather than replacing or rebuilding it.

# Project Context

The generated prompt should begin with instructions similar to:

```text
You are an expert full-stack software engineer.

Your first responsibility is to inspect the existing project.

Do not begin coding immediately.

Understand the project architecture before making changes.

Reuse the existing routing system.

Reuse existing services.

Reuse existing state management.

Reuse existing UI components.

Avoid introducing duplicate logic.

Implement only the requested integration.
```

* Routing structure
* Component hierarchy
* Existing API layer

* Files to inspect
* Files to modify
* Files to create

* Existing reusable utilities

* Testing strategy

Only after the plan is complete should implementation begin.

# Acceptance Criteria

The Windsurf prompt is complete when

✓ It is dynamically generated using the user's selections.

✓ It contains the correct API configuration.

✓ It explains authentication.

✓ It explains rendering strategy.

✓ It explains pagination, filtering and search.

✓ It instructs Windsurf to inspect the project before implementation.

✓ It promotes reuse of existing architecture.

✓ It avoids duplicate code.

✓ It includes a validation checklist.

✓ It can be copied directly into Windsurf and executed without requiring additional instructions.
