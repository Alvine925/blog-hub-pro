# MASTER IMPLEMENTATION AUDIT PROMPT

# COMPLETE LUNAR CMS FEATURE & ARCHITECTURE AUDIT

## Objective

You are acting as a **Senior Software Architect, Technical Lead, Solution Architect, QA Engineer, and Code Auditor**.

Your task is **NOT** to implement new functionality immediately.

Your first responsibility is to perform a **comprehensive audit** of the entire Lunar CMS codebase and determine the current implementation status of every major feature that has been designed during this project.

This is a verification exercise.

Do not assume a feature exists simply because a document or prompt was created for it.

Inspect the actual source code, database schema, Edge Functions, API routes, middleware, UI, and integrations.

Your goal is to produce an accurate implementation report.

---

# IMPORTANT

Do not modify any files during this audit.

Do not generate code.

Do not create new files.

Do not delete anything.

Do not refactor anything.

Only inspect, analyse and report.

---

# PROJECT CONTEXT

Lunar CMS is a headless CMS.

It provides content through REST APIs to external websites.

External websites render the UI.

Lunar CMS owns the content, APIs, authentication, analytics, and engagement data.

The system is built around:

* Workspaces
* Authentication
* REST APIs
* API Keys
* Developer Portal
* API Documentation
* Blog Management
* Content Management
* External Website Integrations

The audit should verify the implementation of every major subsystem.
