# PROMPT – IMPLEMENT SOCIAL SHARING SUPPORT IN LUNAR CMS REST API

## Objective

Extend the Lunar CMS REST API to fully support social sharing for all content types.

This functionality is **not responsible for rendering the share buttons or modal**. Those belong to the client website.

Instead, Lunar CMS should expose all the metadata required for client websites to easily implement professional social sharing.

This feature must work for:

* Blog Posts
* Articles
* News
* Products
* Future content types

The implementation should be API-first and automatically available through all REST endpoints.

---

# IMPORTANT ARCHITECTURE

Lunar CMS is the Content Provider.

The client website is responsible for:

* Rendering the page
* Rendering share buttons
* Opening the share modal
* Setting page metadata
* Sharing its own URL

Lunar CMS is responsible for providing all metadata required to accomplish this.

The API should never generate Facebook or LinkedIn URLs.

The API should never open share dialogs.

The API should never return links pointing back to Lunar CMS.

The shared URL must always be the client's website URL.

---

# IMPLEMENT SHARE METADATA

Every content response should expose a dedicated object named:

```json
social
```

Example

```json
{
  "social": {
    "title": "...",
    "description": "...",
    "image": "...",
    "alt": "...",
    "hashtags": [],
    "twitterCard": "summary_large_image",
    "type": "article"
  }
}
```

The values should be generated using the following priority.

Title

1. social_title
2. seo_title
3. title

Description

1. social_description
2. seo_description
3. excerpt
4. short description

Image

1. social_image
2. featured_image
3. cover_image

Alt Text

Use the featured image alt text.

Twitter Card

Default

```
summary_large_image
```

Open Graph Type

Blog

```
article
```

Products

```
product
```

News

```
article
```

Articles

```
article
```

FAQs

```
website
```
