# IMPLEMENTATION PROMPT

# EXTEND LUNAR CMS INTO A COMPLETE HEADLESS CONTENT PLATFORM

## Objective

Lunar CMS currently has a mature Blog module with REST APIs, developer tools, workspaces, authentication, API Keys, AI generation, and content management.

The next phase is to evolve Lunar CMS from a **Blog CMS** into a **complete Headless Content Platform**.

This must be done by extending the existing architecture, **not** by creating separate systems.

All new content types must integrate seamlessly with the existing architecture and behave exactly like the Blog module.

---

# IMPORTANT

Before writing any code, inspect the existing project.

The following systems already exist and must be reused:

* Authentication
* Workspaces
* API Keys
* REST API
* API Gateway
* Content Router
* Developer Portal
* API Documentation
* API Explorer
* AI Content Generation
* Rich Text Editor
* Media Library
* SEO System
* Search
* Categories
* Tags
* Slug Generation
* Publishing Workflow
* Drafts
* Scheduling
* Analytics
* Feature Flags
* Workspace Settings

Do not duplicate any of these systems.

Everything should extend the existing architecture.

---

# OBJECTIVE

Lunar CMS should support multiple content types.

Instead of only

```text
Blogs
```

The CMS should become

```text
Content

├── Blogs
├── Articles
├── News
├── Products
├── FAQs
├── Pages
```

All of these must behave consistently.

---

# PHASE 1 — CONTENT ARCHITECTURE

Review the existing Blog architecture.

Use it as the blueprint.

Every new content type should inherit the same capabilities wherever applicable.

Do not copy and paste code.

Refactor shared functionality into reusable services if necessary.

# PHASE 2 — NEW CONTENT TYPES

Implement support for the following content types.

## 1. Articles

Purpose

General long-form content.

Examples

* Guides
* Tutorials
* Case Studies
* Documentation
* Educational Content

Capabilities

* Rich Text
* Categories
* Tags
* Featured Image
* Author
* SEO
* Slug
* Draft
* Scheduled Publishing
* Analytics
* Search
* Related Articles
* AI Generation

REST API

```http
GET    /v1/articles
GET    /v1/articles/{slug}
POST   /v1/articles
PUT    /v1/articles/{id}
DELETE /v1/articles/{id}
```

---

## 2. News

Purpose

Time-sensitive updates.

Examples

* Company News
* Press Releases
* Product Updates
* Announcements

Capabilities

* Featured Image
* Categories
* SEO
* Publish Date
* Expiry Date (optional)
* Breaking News Flag
* Featured News
* Related News

REST API

```http
GET    /v1/news
GET    /v1/news/{slug}
POST   /v1/news
PUT    /v1/news/{id}
DELETE /v1/news/{id}
```

---

## 3. Products

Purpose

Product catalog management.

This is **not** an e-commerce checkout system.

It is a content management system for products.

Support

* Product Name
* Slug
* Description
* Rich Text
* Gallery
* Featured Image
* Categories
* SKU
* Brand
* Price
* Compare Price
* Currency
* Product Status
* Specifications
* Features
* Downloads
* SEO
* Related Products

REST API

```http
GET    /v1/products
GET    /v1/products/{slug}
POST   /v1/products
PUT    /v1/products/{id}
DELETE /v1/products/{id}
```

Future-ready for integrations with external commerce platforms.

---

## 4. FAQs

Purpose

Frequently Asked Questions.

Each FAQ should contain

* Question
* Answer
* Category
* Sort Order
* Status
* SEO (optional)

Support

* Categories
* Search
* Featured FAQs
* Ordering

REST API

```http
GET    /v1/faqs
GET    /v1/faqs/{id}
POST   /v1/faqs
PUT    /v1/faqs/{id}
DELETE /v1/faqs/{id}
```
