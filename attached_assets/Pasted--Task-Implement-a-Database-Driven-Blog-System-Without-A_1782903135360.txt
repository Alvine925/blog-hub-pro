# Task: Implement a Database-Driven Blog System Without Affecting the Existing Blog Functionality

## Objective

Implement a new database-driven blog system for the project.

**Important constraints:**

* Do **NOT** modify or break the existing blog functionality.
* Do **NOT** replace the existing blog pages.
* Do **NOT** remove any current routes.
* This should be implemented as a completely separate feature with its own routes and components.
* Existing code should continue working exactly as it does today.

---

# Overall Architecture

There are two applications in this project:

### 1. Admin Application

This is where administrators manage content.

Add a brand-new sidebar/menu item called:

**Blogs**

Administrators should be able to:

* View blog posts
* Create blog posts
* Edit blog posts
* Delete blog posts
* Save drafts
* Publish posts
* Schedule posts (optional if easy to implement)

The admin should save all blog posts into Supabase.

---

### 2. Landing Application

The Landing folder is a Next.js application.

It should fetch published blog posts from Supabase and display them.

This should be implemented as **new routes only**.

Do not modify the existing blog implementation.

---

# Required Database

Create a new table named:

blog_posts

Recommended columns:

```sql
id UUID PRIMARY KEY

title TEXT

slug TEXT UNIQUE

excerpt TEXT

content TEXT

cover_image TEXT

category TEXT

tags JSONB

author_name TEXT

seo_title TEXT

meta_description TEXT

featured BOOLEAN DEFAULT FALSE

status TEXT

published_at TIMESTAMP

reading_time INTEGER

views INTEGER DEFAULT 0

created_at TIMESTAMP

updated_at TIMESTAMP
```

Status values:

* draft
* published
