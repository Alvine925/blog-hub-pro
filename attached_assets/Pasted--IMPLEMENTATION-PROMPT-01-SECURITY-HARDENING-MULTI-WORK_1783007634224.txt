# TASK 1 — REMOVE CLIENT-SIDE SERVICE ROLE EXPOSURE

The audit identified that a Service Role key is exposed or used in client-accessible code.

This is a critical security issue.

Inspect the project.

Verify

* Environment Variables
* Frontend
* Admin Dashboard
* Next.js Client Components
* React Components
* API Calls
* Supabase Client Initialisation

The Service Role Key must never be exposed to the browser.

Implement the following rules.

• Service Role Key may only exist inside secure Edge Functions or server-side execution environments.

• All client requests requiring privileged operations must go through authenticated server endpoints or Edge Functions.

• Replace any direct Service Role usage from the frontend.

• Verify no secrets are bundled into production builds.

# TASK 2 — VERIFY COMPLETE WORKSPACE ISOLATION

Audit every API endpoint.

Audit every database query.

Audit every Edge Function.

Audit every repository.

Every operation must automatically resolve the workspace.

Never trust values sent by the client.

The client must never determine

* workspace_id
* owner_id
* tenant_id

Every request must derive the workspace from authentication or API Key validation.

Verify

Blogs

Pages

Categories

Media

Settings

Comments

Likes

Views

Analytics

API Keys

Uploads

Search

Everything must be isolated per workspace.

# TASK 5 — CONSOLIDATE REST API ROUTING

The audit identified two routing systems.

* api-gateway
* content-router

This creates duplicated logic.

Inspect both.

Determine which architecture is the primary implementation.

Deprecate the legacy route handling.

The project must have ONE unified request lifecycle.

Every endpoint must follow

Incoming Request

↓

Authentication

↓

API Key Validation

↓

Workspace Resolution

↓

Permission Validation

↓

Validation

↓

Business Logic

↓

Content Transformer

↓

Response Builder

↓

JSON Response

No endpoint should bypass this flow.
