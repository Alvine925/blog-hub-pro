# IMPLEMENTATION PROMPT 03C

# LOVEABLE AI INTEGRATION PROMPT GENERATOR

## Objective

Implement a Loveable-specific prompt generator that creates optimized prompts for Loveable projects.

Unlike Cursor, Loveable frequently works at the application level and can generate large portions of functionality. Therefore prompts generated for Loveable should be more prescriptive and provide clear implementation requirements.

The goal is for a user to:

```text
Generate API Key

↓

Open Integration Center

↓

Select Loveable

↓

Select Framework

↓

Select Content Types

↓

Generate Prompt

↓

Copy Prompt

↓

Paste Into Loveable

↓

Loveable Integrates Lunar CMS
```

---

# Existing CMS

The following already exist.

DO NOT rebuild.

* Authentication
* Workspaces
* API Keys
* REST API
* Blogs
* Pages
* Events
* Categories
* Tags
* Media
* Dashboard
* Existing UI

The generated prompt must tell Loveable to integrate with existing systems only.


---

# Authentication Requirements

All requests should include

```http
Authorization:
Bearer API_KEY
```

Publishable Keys only.

Never place Secret Keys in frontend projects.

Never hardcode keys.

Use environment variables.

# Acceptance Criteria

The generated Loveable prompt is complete when

✓ It is customized for the selected framework.

✓ It includes the correct API configuration.

✓ It explains authentication.

✓ It explains pagination.

✓ It explains search.

✓ It explains SEO.

✓ It instructs Loveable to inspect the existing project before coding.

✓ It reuses existing architecture.

✓ It avoids duplicate functionality.

✓ It can be pasted directly into Loveable without requiring additional instructions.

✓ It produces production-ready integration code.
